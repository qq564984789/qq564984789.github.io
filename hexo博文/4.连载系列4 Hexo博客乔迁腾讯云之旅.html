连载系列4 狡兔三窟：从Github到腾讯云服务器

为什么又要乔迁？

以前我们出门有辆自行车就可以了，后来嫌慢就换摩托车，再后来为了避免日晒雨淋就做轿车里了。不难发现，大家口袋里钢镚一多，要求就越来越高。同样地，为了提升我们博客的访问速度和独立性，我们准备把它从一个巨大的免费公共平台Github搬到也不赖的收费私有平台腾讯云服务器上。收费有收费的道理，因为免费的东西虽然不收费，但是服务较差。

乔迁策略？

上一节的将Hexo博客乔迁Github，从实现原理来说，就是将本地博客根目录（blog\）下public文件夹里的所有文件上传至Github中的某个特定仓库，然后使用该仓库的Pages功能展示博客页面。那么这一节我们也是从本地出发，只是把目的地变成腾讯云服务器吗？不是！我们的策略是：将Github作为中间者，本地博客有更新就上传到Github, 而我们的云服务器从Github获取那个特定仓库的内容，然后把它显示出来。这就类似你的货（本地博客）通过快递员（Github）到买家（云服务器）那里。

动手实验

一、云服务器的准备

  1. 登录腾讯云：用微信扫一扫即可
  2. 购买服务器实例:操作系统选择Ubuntu, 版本是Server18.04.1 LTS。

注意仓库名称必须写为username.github.io 其中username是你注册Github使用的用户名。
3. 登录服务器：登录时用户名默认为ubutntu,你需要设置初始密码。

二、宝塔面板的使用

  1. 安装宝塔面板（Bt-Panel）：进入服务器系统后，执行以下命令
wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh

最后如果出现Congratulations!字样，表示安装成功。
2. ** 访问面板：在浏览器中输入上图最中Bt-Panel:后的网址(我这里是http://106.54.19.80:8888/)，这个网址其实就是我们服务器的公网IP，而面板的默认端口是8888。然后输入数图中的用户名(tfpk6oif)和密码。

三、从Github获取自己的博客

  • 将当前路径切换到网站站点所在目录
cd /www/wwwroot/
  • 把Github上的博客站点文件拷贝到当前目录下
git clone https://github.com/qq564984789/qq564984789.github.io.git
  • 将qq564984789.github.io.git下的所有内容拷贝至yangkai123.top下
cp -rf qq564984789.github.io/* yangkai123.top
  • 观察下yangkai123.top中的内容，是否和你预想的一样呢？
cd yangkai123.top

git reset --hard origin/master
