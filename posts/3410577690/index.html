连载系列3 Hexo博客乔迁Github之旅

Hexo Github
Hexo
发布日期:   2020-02-09
文章字数:   873
阅读次数:  

为什么要乔迁？

前两节讲述的是Hexo博客在本地的搭建，这时候博客是部署在你自己电脑上的，也只有用你的电脑才能访问这个博客，我把它叫本地博客。在网络时代，我们都希望自己的博客可以联网访问。我所谓的网络博客，就是将博客部署在一台具有公网IP地址的电脑上(通常为服务器)，这样无论何时何地，任何人都可以访问，它俨然就成了你的个人网站。

乔迁去哪儿？

  • 别人的服务器
    • Github:全球最大的开源项目网站,提供代码托管服务，含有的Pages功能可以展示网页。现已被微软公司收购，服务器在美国，连接速度慢。
    • Coding:中国版Github，优点是连接速度快，但是山寨。
  • 自己的服务器
    • 腾讯云:腾讯管它叫CVM，价格便宜，活动时1核2G的，1M带宽的入门版新用户年费是128元。
    • 阿里云:阿里管它叫ECS，价格较高，一般开发者只能等双十一了。

如果你只是偶尔写一下博客，则可以使用Github；希望长期撰写博客的朋友，建议购买云服务器；本篇只介绍如何将Hexo博客乔迁Github。

到底乔迁什么？

所谓将Hexo博客乔迁Github，从实现原理来说，就是将本地博客根目录（blog\）下public文件夹里的所有文件上传至Github中的某个特定仓库，然后使用该仓库的Pages功能展示博客页面。

动手实验

一、Github的准备

  1. 注册Github：右上角的sign in是登录，sign up是注册。
  2. 新建仓库(Repository):点击create repository按钮。

注意仓库名称必须写为username.github.io 其中username是你注册Github使用的用户名。

  1. 查看仓库配置：点击仓库右上角的Settings选项。

如果发现Github Pages这一部分有Your site is published at https://username.github.io字样，则表明仓库具备了网页展示功能。提醒下：这个新建的仓库就是用来下面保存Hexo博客的各种资源文件的。

二、绑定Hexo和Github

  1. 修改本地站点配置文件: 打开站点目录（blog\）下的_config.yml文件，将最后一部分修改为
    deploy:  
          type:  git
          repo:  git@github.com:username/username.github.io.git
          branch:  master

这段源码的意思是：我们将采用git工具来做Hexo博客的部署，部署在哪里呢？位置在Github上的username/username.github.io.git这样一个仓库里，且是仓库的master分支上。注意:上述冒号的后面都有一个空格。

  1. 安装关联hexo和git的工具: 
npm install hexo-deployer-git –save

如果最后可以看到这个工具的版本号，则表明安装成功。

三、Hexo升天

  1. 部署Hexo：执行以下命令
hexo d

这个命令会将本地博客根目录（blog\）下public文件夹里的所有文件上传至Github上的username/username.github.io.git这个仓库。d:depoly 是部署的意思。

  1. 博客预览：在浏览器中输入https://username.github.io,如果看到新页面则大功告成。

如果以后写了新的博文后希望更新到Github上的博客,则必须先执行清除本地旧数据（网页）和生成新数据（网页）两个动作、意即依次执行hexo clean和hexo g操作。最后再执行部署动作hexo d。
