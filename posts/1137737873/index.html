网页和BLE设备互动

BLE 网页
BLE
发布日期:   2020-02-15
文章字数:   2.9k
阅读次数:  

如果告诉你一个网页可以和你周边的蓝牙设备安全而且极具隐私性地通信，你会相信吗？告诉你吧，心率计、音乐灯这些东西都可以和网页直接互动。

目前为止，只有本地手机APP可以和蓝牙设备互动。网页蓝牙（ Web Bluetooth ）API就是为了改变现状而生，已经开始支持网页浏览器了。实际上早年Google的 Physical Web项目，人们就已经可以直接通过网页和物理设备通信了。

一、开始之前

这篇文章假设你已经有了解了低功耗蓝牙(BLE)和GATT是如何工作的。

虽然网页蓝牙规范（ Web Bluetooth API specification）还没有完成，但是Chrome团队希望极客们（我说的就是你）可以来尝尝鲜，试试还在推进中的API，然后给API的规范或者实现细节一些反馈。

网页蓝牙API的部分功能已经在Chrome操作系统，安卓、Mac、Windows上的Chrome浏览器上实现了。也就是说你应该可以实现以下功能了：

  • 请求并连接到附近的蓝牙设备
  • 读/写蓝牙特性（characteristic）
  • 读写蓝牙描述符（descriptor）
  • 接收GATT通知
  • 获取蓝牙设备连接/断开状态

对于早期版本的Winodws和Linux,你还需要到chrome://flags/#enable-experimental-web-platform-features使能高亮标志，然后重启Chrome。

二、安全性要求

为了理解安全性上的折衷，我建议你看下这篇文章 网页蓝牙安全模型，它来自Chrome团队一位名叫Jeffrey Yasskin软件工程师的手笔，主要研究网页蓝牙的API规范。

1、只允许HTTPS访问

网页蓝牙API是Web中新添加的一项功能强大的特性，目前还处于实验/推进阶段，所以Goole的Chrome团队希望该特性只能在安全的上下文环境（secure contexts）中使用。

我们非常关心安全性，所以你会发现新的Web功能都要求HTTPS。网页蓝牙API也不例外，这也是使用HTTPS运行网页的另一个原因。

在开发过程中为了和网页蓝牙互动，你将可以通过http://localhost这个网址外加 Chrome Dev Editor 编辑器、执行python -m SimpleHTTPServer命令。但如果你想部署它，你需要在服务器上启动HTTPS服务。我个人建议使用 GitHub Pages 来做Demo展示。

在服务器上添加HTTPS服务需要获得一个TLS证书，然后再启用它。 请通过这里了解。更多消息，请看这里，你现在可以获取带有新Certificate Authority的免费TLS证书。

2、用户手势要求

因为有安全性要求，名为navigator.bluetooth.requestDevice的函数（功能为发现蓝牙设备），必须由用户的手势来触发，比如触摸屏幕、点击鼠标。我们来谈一谈释放、单击、触摸事件。

button.addEventListener('pointerup', function(event) {
  // Call navigator.bluetooth.requestDevice
});

三、研究代码

网页蓝牙API严重依赖Javascript里的 Promises。如果你还不熟悉，请看这两份材料：

  • Promises tutorial
  • 理解JavaScript中的Promise

1、 请求（访问）蓝牙设备

当前版本的网页蓝牙API规范允许网站运行在中央角色（Central role），然后通过BLE去连接GATT服务器。它支持和蓝牙4.0及以上的设备通信。
当网页使用navigator.bluetooth.requestDevice这个函数去请求访问周边设备时，Google Chrome浏览器会给用户弹出一个设备选择器的提示，你可以选择某个设备或者直接取消访问蓝牙设备的请求。

navigator.bluetooth.requestDevice这个函数必须强制性地带有一种过滤器的对象。过滤器用来返回那些和特定蓝牙设备名字或GATT服务相同的设备。

例如，我现在想要请求的蓝牙设备正在广播电量服务,那么可以写成这样

navigator.bluetooth.requestDevice({ filters: [{ services: ['battery_service'] }] })
.then(device => { /* ... */ })
.catch(error => { console.log(error); });

如果你的蓝牙GATT服务不在官方GATT服务之列，你可以提供UUID全称或16、32位简写。

navigator.bluetooth.requestDevice({
  filters: [{
    services: [0x1234, 0x12345678, '99999999-0000-1000-8000-00805f9b34fb']
  }]
})
.then(device => { /* ... */ })
.catch(error => { console.log(error); });

当然，你也可以基于蓝牙设备的名称去请求访问，例如使用全称过滤器关键字name、名称前缀过滤器关键字namePrefix。这种情形仍需注意，你还是要定义一个optionalServices关键字以便能访问某些服务。如果你没做，当你后面要访问这些服务时会出错。

navigator.bluetooth.requestDevice({
  filters: [{
    name: 'Francois robot'
  }],
  optionalServices: ['battery_service']
})
.then(device => { /* ... */ })
.catch(error => { console.log(error); });

最后，除了关键字filters,你还可以使用acceptAllDevices来显示周边所有的蓝牙设备。你还是要定义一个optionalServices关键字以便能访问某些服务。如果你没做，当你后面要访问这些服务时会出错。

navigator.bluetooth.requestDevice({
  acceptAllDevices: true,
  optionalServices: ['battery_service']
})
.then(device => { /* ... */ })
.catch(error => { console.log(error); });

请注意，上述代码可能导致设备选择器中列出了一堆相关设备，无过滤器由此带来了能耗。谨慎使用该功能。

2、连接到蓝牙设备

如果你从navigator.bluetooth.requestDevice's返回得到一个蓝牙设备，然后准备做什么？让我们连接这个设备——它是一个定义了服务和特性的GATT服务器。

navigator.bluetooth.requestDevice({ filters: [{ services: ['battery_service'] }] })
.then(device => {
  // Human-readable name of the device.
  console.log(device.name);

  // Attempts to connect to remote GATT Server.
  return device.gatt.connect();
})
.then(server => { /* ... */ })
.catch(error => { console.log(error); });

3、读取蓝牙特性值

现在我们已经连接到了一个GATT服务器，然后希望获取一个主服务（Primary GATT Service）和一个属于该服务的特性。让我来试试看，例如去读取当前设备的电量。

在这个例子里，battery_level是官方电量特性。

navigator.bluetooth.requestDevice({ filters: [{ services: ['battery_service'] }] })
.then(device => device.gatt.connect())
.then(server => {
  // Getting Battery Service...
  return server.getPrimaryService('battery_service');
})
.then(service => {
  // Getting Battery Level Characteristic...
  return service.getCharacteristic('battery_level');
})
.then(characteristic => {
  // Reading Battery Level...
  return characteristic.readValue();
})
.then(value => {
  console.log('Battery percentage is ' + value.getUint8(0));
})
.catch(error => { console.log(error); });

如果你使用自定义的GATT特性，你需要提供UUID全称或16、32位简写给service.getCharacteristic。注意到，你同样可以针对特性添加一个characteristicvaluechanged事件监听器来读取特性值。请看这篇文章来理解如何处理另一端设备发出的GATT通知。

.then(characteristic => {
  // Set up event listener for when characteristic value changes.
  characteristic.addEventListener('characteristicvaluechanged',
                                  handleBatteryLevelChanged);
  // Reading Battery Level...
  return characteristic.readValue();
})
.catch(error => { console.log(error); });

function handleBatteryLevelChanged(event) {
  let batteryLevel = event.target.value.getUint8(0);
  console.log('Battery percentage is ' + batteryLevel);
}

4、写一个特性值

写特性和读特性一样简单。这次，让我们使用心率控制点（Heart Rate Control Point）这一特性来复位心率计上电量拓展域的值为0。

我保证此处没有什么奇特的地方。

   navigator.bluetooth.requestDevice({ filters: [{ services: ['heart_rate'] }] })
   .then(device => device.gatt.connect())
   .then(server => server.getPrimaryService('heart_rate'))
   .then(service => service.getCharacteristic('heart_rate_control_point'))
   .then(characteristic => {
     // Writing 1 is the signal to reset energy expended.
     var resetEnergyExpended = Uint8Array.of(1);
     return characteristic.writeValue(resetEnergyExpended);
   })
   .then(_ => {
     console.log('Energy expended has been reset.');
   })
   .catch(error => { console.log(error); });

5、接收GATT通知（Notification）

现在，让我们看看当心率计上的心率测量特性值改变时，网页蓝牙如何知晓？

   navigator.bluetooth.requestDevice({ filters: [{ services: ['heart_rate'] }] })
   .then(device => device.gatt.connect())
   .then(server => server.getPrimaryService('heart_rate'))
   .then(service => service.getCharacteristic('heart_rate_measurement'))
   .then(characteristic => characteristic.startNotifications())
   .then(characteristic => {
     characteristic.addEventListener('characteristicvaluechanged',
                                     handleCharacteristicValueChanged);
     console.log('Notifications have been started.');
   })
   .catch(error => { console.log(error); });

   function handleCharacteristicValueChanged(event) {
     var value = event.target.value;
     console.log('Received ' + value);
     // TODO: Parse Heart Rate Measurement value.
     // See https://github.com/WebBluetoothCG/demos/blob/gh-pages/heart-rate-sensor/heartRateSensor.js
   }

这个例程 展示了如何使用stopNotifications()函数停止接收通知，以及正确地移除和添加characteristicvaluechanged事件监听器。

6、断开连接

为了提供更好的用户体验，当你的网页蓝牙和设备断开连接后会显示一条警告信息，请求你重新连接。

   navigator.bluetooth.requestDevice({ filters: [{ name: 'Francois robot' }] })
   .then(device => {
     // Set up event listener for when device gets disconnected.
     device.addEventListener('gattserverdisconnected', onDisconnected);

     // Attempts to connect to remote GATT Server.
     return device.gatt.connect();
   })
   .then(server => { /* ... */ })
   .catch(error => { console.log(error); });

   function onDisconnected(event) {
     let device = event.target;
     console.log('Device ' + device.name + ' is disconnected.');
   }

你也可以调用device.gatt.disconnect()函数来断开网页和蓝牙设备的连接。这会触发gattserverdisconnected事件监听器。详细信息，请看设备断开连接例程和自动重连例程。

警告：当设备断开连接后，蓝牙GATT属性、服务、特性等都是错误值。这意味着在重新建立连接后，你的代码需要通过getPrimaryService(s), getCharacteristic(s)去重新获取值。

7、读/写蓝牙描述符（descriptor）

蓝牙GATT描述符是用来描述特性值的。你可以像读/写蓝牙GATT特性一样去读/写它们。

例如，让我们来看看如何读取健康测量仪上测量精度的描述符。

在下面这个例子里，health_thermomete是健康测量仪服务、measurement_interval是测量精度特性、gatt.characteristic_user_description是特性的用户描述符。

   navigator.bluetooth.requestDevice({ filters: [{ services: ['health_thermometer'] }] })
   .then(device => device.gatt.connect())
   .then(server => server.getPrimaryService('health_thermometer'))
   .then(service => service.getCharacteristic('measurement_interval'))
   .then(characteristic => characteristic.getDescriptor('gatt.characteristic_user_description'))
   .then(descriptor => descriptor.readValue())
   .then(value => {
     let decoder = new TextDecoder('utf-8');
     console.log('User Description: ' + decoder.decode(value));
   })
   .catch(error => { console.log(error); });

现在你可以读取健康测量仪上测量精度的描述符了，让我们再看看如何更新或改写它。

   navigator.bluetooth.requestDevice({ filters: [{ services: ['health_thermometer'] }] })
   .then(device => device.gatt.connect())
   .then(server => server.getPrimaryService('health_thermometer'))
   .then(service => service.getCharacteristic('measurement_interval'))
   .then(characteristic => characteristic.getDescriptor('gatt.characteristic_user_description'))
   .then(descriptor => {
     let encoder = new TextEncoder('utf-8');
     let userDescription = encoder.encode('Defines the time between measurements.');
     return descriptor.writeValue(userDescription);
   })
   .catch(error => { console.log(error); });

四、例程

所有下面的网页蓝牙例程已经成功验证过。为了最大程度地利用这些资源，我建议你安装安卓上的蓝牙外设模拟器。它可以模拟一个含有电池、心率计、健康测量仪等服务的蓝牙外设。

1、初学者

  • Device Info：获取BLE的基本设备信息。
  • Battery Level：从广播电池信息的BLE设备中获取电池信息
  • Reset Energy 将心率计中的能量消耗值复位
  • Characteristic Properties ：显示BLE设备里某个特性的所有属性
  • Notifications ：启动/停止特性通知
  • Device Disconnect：断开连接并获得通知
  • Get Characteristics ：获取某个服务的所有特性
  • Get Descriptors：获取某个服务中所有特性的描述符。
